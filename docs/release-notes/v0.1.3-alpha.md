# Cliniq-OX: Release Notes `v0.1.3-alpha`

This document details the modifications, API changes, and database migrations introduced in version `v0.1.3-alpha`.

---

## 1. Summary of Changes
- **OT Rooms Master:** Added `ot_rooms_master` configuration for scheduling and conflict checking.
- **Doctor Profiles:** Decoupled `doctors` profile table from authorization data.
- **Estimate Status Workflow:** Added `DRAFT`, `APPROVED`, `LOCKED`, and `CANCELLED` statuses. Modifying approved/locked estimates forces version snapshot generation.
- **Multi-Hospital Readiness:** Introduced `hospital_id` to partition data for multi-hospital deployments.
- **Permission Matrix:** Generated `docs/permission-matrix.md` mapping role-based permissions (Receptionist, Doctor, Admin).
- **Document Generation Engine:** Configured PDF generation schemas for Estimates, Invoices, Receipts, Consent forms, and Discharge summaries.
- **Enhanced Audits & Soft-Deletes:** Appended `user_name_snapshot` and `user_role_snapshot` to audit logs, and extended soft-delete parameters across all transactional tables.
- **Backup & Rollback Procedures:** Established guidelines for Change Proposals, rollback instructions, and versioned backups (`backups/vX.X.X/`).

---

## 2. Refinements Applied
- **Date of Birth:** Added `date_of_birth` (DATE) to `patients` table, treating age as derived logic.
- **Patient Gender ENUM:** Converted patient gender column into a strict ENUM ('MALE', 'FEMALE', 'OTHER').
- **Estimate Approvals:** Appended `approved_by` and `approved_at` to `estimates` table.
- **Calendar Event Auditing:** Added `created_by`, `approved_by`, `approved_at`, and `event_status` to `calendar_events` table.
- **Immutable Estimate Numbering:** Documented ADR 015 sequential immutable coding structures.
- **Document Engine Reservation:** Mapped hooks for future `patient_documents` module integration.
- **Template Item Types:** Appended `item_type` ENUM to `estimate_template_items`.
- **Multi-Surgeon Recommendation:** Formulated ADR 016 mapping future assistant surgeon integrations.
