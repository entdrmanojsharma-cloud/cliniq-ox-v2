# Cliniq-OX: Release Notes `v0.1.8-alpha`

This document details the modifications, API changes, and database migrations introduced in version `v0.1.8-alpha`.

---

## 1. Summary of Changes
- **Document Generations Table:** Established the `document_generations` table to archive generated PDFs (Estimates, Invoices, Receipts, Consent Forms) and metadata history. This serves as the single source of truth and allows unlimited document revisions per entity.
- **Obsolete Column Removal:** Dropped `generated_file_name`, `generated_at`, and `generated_by` columns from the `estimates` table.
- **Estimate Template Refactoring:** Modularized the monolithic `estimate.js` print template into a dedicated folder (`backend/modules/documents/templates/estimate/`) comprising 7 separate sub-components: `header`, `patient-section`, `surgery-section`, `charges-section`, `summary-section`, `footer`, and `index`.
- **Standards Compliance:** Confirmed that all newly created files strictly remain under the 300-line limit.

---

## 2. Refinements Applied
- **Archival Logging Hook:** Refactored Documents service and repository layers to write execution records into the new `document_generations` table for all generated files.
- **Unit Testing Suite updates:** Updated repository mock parameters and assertions inside `tests/unit/test_documents.js` to assert table entries.
- **Calculations Preservation:** Ensured all discounts (percentage/fixed, component-level, and global), item groups, and taxability/print flags are fully respected by the templates.
